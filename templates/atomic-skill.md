# Atomic template

Purpose: one focused capability.

Use when: the request matches this capability.

Inputs:
- goal
- context
- limits

Steps:
1. Read the request.
2. Pick the relevant method.
3. Apply the method.
4. Return the result.
5. Check the result.

Result:
Define the expected answer or file.

References:
List the supporting sections.

Checks:
- one purpose
- clear steps
- useful result
