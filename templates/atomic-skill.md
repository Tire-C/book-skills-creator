# Atomic template

Purpose: one focused capability.

Use when: the user request matches this capability and does not need a larger workflow.

Inputs:
- goal
- context
- limits
- preferred output

Steps:
1. Restate the goal.
2. Choose the relevant method.
3. Apply the method step by step.
4. Return the result.
5. Check the result against the limits.

Result:
Define the expected answer, file, plan, table, or decision.

References:
List the supporting source sections or reference files.

Checks:
- one purpose
- clear inputs
- clear steps
- useful result
- no overlap with another unit
